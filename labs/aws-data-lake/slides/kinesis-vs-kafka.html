Kinesis Data Streams vs Firehose

Data Streams

A durable, replayable log you read from. You build the consumer — Lambda, KCL, or Managed Flink sits on top.

Firehose

A managed delivery service to AWS sinks. No consumer code — point at a destination and data flows.

Dimension Kinesis Data Streams Kinesis Firehose
Mental model "Pipe you read from" "Delivery service"
You write consumer code? Yes — Lambda / KCL / Flink No — Firehose delivers
Latency Sub-second ~60 seconds (buffer)
Retention 24h default, up to 365 days None — forwarded then forgotten
Throughput model Shards — you provision (1 MB/s in, 2 MB/s out per shard) Auto-scaling — no capacity planning
Destinations Wherever your consumer writes S3, Redshift, OpenSearch, Splunk, Datadog, HTTP
Multi-consumer Yes No
Pricing Per-shard-hour + per-PUT Per-GB ingested
Mnemonic: Streams = stream (you build on top). Firehose = hose (point at a sink, water flows).

The Full Ecosystem Mapping

Kafka / OSS AWS Equivalent What It Does
Kafka brokers + topics Kinesis Data Streams The durable log
Self-managed Kafka Amazon MSK Managed Kafka with real Kafka API

Where the Analogies Leak — and When to Pick Each

Aspect Kafka Kinesis Data Streams
Throughput per partition/shard Hardware-limited (10+ MB/s common) Hard caps: 1 MB/s write, 2 MB/s read per shard
Exactly-once semantics Supported via transactional producers At-least-once only — build idempotency yourself
Compression Per-topic config (gzip / snappy / zstd / lz4) Up to your producer code; no topic-level setting
Cross-region replication MirrorMaker / Confluent Replicator Not built-in — write a Lambda fan-out
Operations cost You run brokers (or pay for MSK / Confluent) Zero ops, but locked to AWS region / account
Client ecosystem Massive open-source library AWS SDKs only; no Kafka-protocol compatibility

When to Pick

  • Kinesis Data Streams: zero broker ops, team deep in AWS, per-shard caps work for your volume
  • Amazon MSK: existing Kafka code, need exactly-once, or Confluent ecosystem (speaks real Kafka protocol)
  • Kinesis Data Firehose: building an S3 sink in Kafka Connect — without operating one
