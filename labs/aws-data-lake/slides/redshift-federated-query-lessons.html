IAM Role — 3 Permissions Needed

  • Secrets Manager — read federated creds
  • Glue Data Catalog — Spectrum metadata
  • S3 GetObject / ListBucket — data reads

Attach to Redshift Namespace, not Workgroup.

Federated Query — How It Works

  • Creates external schema FROM POSTGRES
  • Secret stores read-only Postgres credentials
  • Live query — Aurora bears the read load
  • Read-only; no writes to Aurora possible

Spectrum — How It Works

  • Creates external schema FROM DATA CATALOG
  • Metadata lives in Glue (even without Glue console)
  • DDL defines external table over S3 prefix
  • Billed per TB scanned — partition to cut cost

Hard-Won Lessons

8 Gotchas That Will Break Your Setup

⚠ These cost the most time in the original setup
1

SCRAM vs md5 — The Final Blocker

Aurora defaults to scram-sha-256; Redshift only speaks md5. Run SET password_encryption='md5' and CREATE USER in the same session — a new connection reverts the setting.

2

Never Use the rds!cluster-… Master Secret

AWS auto-rotates it back to SCRAM, re-breaking the connection every rotation. Create a dedicated redshift_fed md5 user with its own secret.

3

Glue Is Required Even Without the Glue Console

CREATE EXTERNAL DATABASE writes to the Glue Data Catalog silently. The IAM role needs glue:CreateDatabase or the schema creation fails.

4

Test the Network From Inside the Redshift VPC

Your laptop and Redshift take different network paths to Aurora. A passing psql from your machine proves nothing about what Redshift can reach. Use openssl s_client from inside the VPC.

5

TLS Fixes Happen on Aurora — Not Redshift

CREATE EXTERNAL SCHEMA … FROM POSTGRES has no SSL knob. All cert and TLS issues are resolved on the Aurora side (CA identifier, SG rules, VPC routing).

6

Keep Everything in the Same Region

Redshift, Aurora, and S3 data must all be in us-west-2. The TICKIT sample lives in us-east-1 — copy it to your bucket first, or pay cross-region latency and egress charges.

7

LOCATION Is a Folder Prefix, Not a File

Must end with / — Spectrum reads every file under the prefix. Pointing at a file path rather than a folder returns 0 rows with no error.

8

Partitioned Tables Need Explicit ADD PARTITION

Spectrum does not auto-discover partitions. Each partition must be registered with ALTER TABLE … ADD PARTITION. No crawler = no automation.
